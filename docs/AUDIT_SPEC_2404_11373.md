# 2404.11373 Audit Spec

This document defines the project-side audit target for a strict Fig.1 reproduction of arXiv:2404.11373. The goal is not "a similar-looking contour", but a reproducible claim level with explicit failure modes.

## Goal

A run can only be called paper-faithful if it satisfies all of:

1. Experiment setup matches the paper.
2. Fig.1 morphology matches the paper and agrees with `pyRing`.
3. Internal calibration is validated with the Appendix B HPD coverage test.

## Required Cases

The audit target includes exactly these three zero-noise injections:

- `kerr220`: `(2,2,0)`
- `kerr221`: `(2,2,0) + (2,2,1)`
- `kerr330`: `(2,2,0) + (3,3,0)`

All three use:

- `M_f = 67 M_sun`
- `chi_f = 0.67`
- `ra = 1.95`
- `dec = -1.27`
- `psi = 0.82`
- `gps_h1 = 1126259462.42323`

Inclination is case-dependent:

- `kerr220 = pi`
- `kerr221 = pi`
- `kerr330 = pi/4`

Target SNR:

- `kerr220 ~= 14`
- `kerr221 ~= 14`
- `kerr330 ~= 53`

## Prior Contract

The Table II prior contract is:

- `M_f in [20, 300] M_sun`
- `chi_f in [0, 0.99]`
- `A_220 in [0.1, 50] x 1e-21`
- `A_221 in [0, 50] x 1e-21`
- `A_330 in [0, 50] x 1e-21`
- `phi_lmn in [0, 2pi]`

The inferred parameter vector is:

- `theta = {M_f, chi_f} union {A_lmn, phi_lmn}_lmn`

Expected dimensions:

- `kerr220`: 4
- `kerr221`: 6
- `kerr330`: 6

## Data Contract

Paper-faithful data preparation requires:

- time-domain ringdown segment
- `2048 Hz`
- `0.1 s`
- `204` bins per detector
- `408` bins after H1/L1 concatenation
- whitening via `PSD -> ACF -> Toeplitz covariance -> Cholesky`

If the figure pipeline does not actually use this whitening path, the result may still be useful, but it is not Appendix-A-faithful.

## Training Contract

Paper-faithful training uses:

- NSF density estimator
- `num_transforms = 5`
- `hidden_features = 150`
- `num_blocks = 2`
- `num_bins = 10`
- `batch_norm = True`
- embedding MLP: `408 -> 150 -> 150 -> 128`
- `batch_size = 512`
- `learning_rate = 1e-3`
- `validation_fraction = 0.1`
- round 1: `50k`
- later rounds: `100k`
- `trunc_quantile = 1e-4`
- `stopping_ratio = 0.8`
- varying noise enabled during training

## TSNPE Contract

Paper-faithful TSNPE requires:

- HPD-style truncation using the previous posterior density threshold
- not a box, percentile, or ellipsoid surrogate
- early stop when `V_t / V_(t-1) > 0.8`
- first-round samples discarded in later rounds
- progressive shrinkage across rounds

Immediate failure conditions:

- `acceptance = 1`
- `volume = 1`
- posterior remains prior-wide
- parameters leave the prior domain

## Fig.1 Contract

For a figure-faithful Fig.1:

- all three cases must be present
- plot must show 2D `68% / 90%` credible contours in `(M_f, chi_f)`
- `pyRing` baseline must be present
- injected truth must be marked
- `kerr330` must be visibly narrower than `kerr220` and `kerr221`
- SBI and `pyRing` contours must substantially overlap

## Coverage Contract

Paper-faithful coverage requires Appendix B, not 1D percentile sanity checks:

- HPD `gamma(theta*, x*)` definition
- test prior equals the final truncated prior
- `100` draws per catalog
- `100` catalogs
- CDFs near the diagonal
- KS p-value summary

Without this, the best available grade is below `A`.

## Audit Grades

The project uses a pragmatic four-level output:

- `A Paper-faithful`
  All implemented paper-faithful checks pass, including Appendix B coverage.
- `B Figure-faithful but diagnostics incomplete`
  Fig.1 and `pyRing` agreement look correct, but Appendix B coverage is still incomplete.
- `C Pipeline-validated only`
  Core mechanics run and posterior shrinkage is real, but paper-faithful requirements are still incomplete.
- `D Invalid for paper comparison`
  TSNPE failed to shrink, samples are invalid, or the artifacts are too incomplete to support a paper comparison.

## Current Automation Scope

The automated audit script checks only what can be verified from code and saved artifacts. It does not upgrade unsupported items to pass by inference.

Current coded checks include:

- injection values and case definitions
- prior ranges and parameter dimensions
- model and TSNPE defaults
- detector-input dimensionality
- exact-whitening implementation presence
- whether the Fig.1 pipeline actually uses exact whitening
- varying-noise implementation presence
- whether the Fig.1 pipeline actually uses varying noise
- posterior shrinkage and sample validity
- all-case presence
- `pyRing` and overlay artifact presence
- coverage-summary artifact presence

Anything not explicitly checked remains unverified, not implicitly passed.
